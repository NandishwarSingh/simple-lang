## Analysis (hand-written)

- **primes: Simple ties C, C++, and Rust (1.00x).** The workload is dominated
  by hardware integer division, so every code generator hits the same wall.
  Proof that QBE's output is fundamentally sound.
- **fib: 1.70x C.** clang/rustc aggressively inline the recursion; QBE
  doesn't inline at all. An inlining pass in our future MIR layer is the fix.
- **collatz: 3.21x C — the best finding of the run.** `n / 2` and `n % 2`
  compile to real `sdiv` instructions under QBE, while clang/rustc turn them
  into shifts and masks. Classic *strength reduction* — exactly the kind of
  optimization the planned MIR stage exists for. This number should drop
  hard in v0.5.
- **Memory: 1.3 MB, identical to C.** No runtime, no collector. Go carries
  4.0 MB before user code allocates a byte.
- **Binary size: 49 KB** vs C 33 KB, Rust 455 KB, Go 2434 KB.
- **Compile speed: 0.05 s** — clang-class, well ahead of rustc and go build.
