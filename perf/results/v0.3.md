## Analysis (hand-written)

**This is the ARC release.** Compare strbuild against v0.2-strbuild-baseline:

- **Memory: 307.9 MB → 1.5 MB.** The leak is gone; Simple now matches C's
  footprint while managing memory automatically. `leaks --atExit` reports
  0 leaked bytes on every example plus a 20k-iteration churn stress test.
- **Speed: 0.12 s → 0.12 s (vs-C ratio improved 1.71x → 1.50x).** All the
  retain/release/free work was paid for by the new string header: O(1)
  `len()` and concat that skips strlen. Automatic memory management arrived
  as a net *win*. Simple is now ~tied with Rust (0.11 s) on this workload.
- **mimalloc experiment (dependency policy):** DYLD-interposed mimalloc
  3.4 on strbuild: 0.13–0.14 s vs 0.12–0.13 s system malloc, same RSS.
  Verdict: **no benefit — dependency rejected.** Apple's system allocator
  is already excellent for this pattern; revisit only if a future workload
  shows allocator pressure.
- fib/primes/collatz: unchanged (ARC touches nothing without strings).
  The standing gaps (fib 1.7x inlining, collatz 3.2x strength reduction)
  remain the MIR layer's job.
